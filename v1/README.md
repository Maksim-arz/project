# Система управления клиентами и заказами

Консольное приложение на Python для ведения базы клиентов и их заказов. Все данные сохраняются в JSON-файл между сессиями.

## Запуск

```bash
python v1/code/project.py
```

Требования: Python 3.x, сторонние библиотеки не нужны.

## Структура проекта

```
project/
├── v1/
│   ├── code/
│   │   └── project.py      # Весь исходный код
│   └── database.json       # База данных (создаётся автоматически)
└── README.md
```

## Главное меню

При запуске открывается интерактивное меню:

```
=== Главное меню ===
1. Добавить клиента
2. Показать клиентов
3. Удалить клиента
4. Показать активные заказы клиента
5. Показать историю заказов клиента
6. Добавить заказ
7. Показать все заказы
8. Изменить статус заказа
9. Удалить заказ
0. Выход
```

## Функции

### Клиенты

| Пункт | Действие | Описание |
|-------|----------|----------|
| 1 | Добавить клиента | Запрашивает имя, телефон и e-mail. Присваивает уникальный ID. |
| 2 | Показать клиентов | Выводит список всех клиентов с их ID и контактами. |
| 3 | Удалить клиента | Удаляет клиента по ID, а также все его заказы. |

### Заказы

| Пункт | Действие | Описание |
|-------|----------|----------|
| 4 | Активные заказы клиента | Заказы со статусом «Новый» или «В работе», отсортированные по сумме (по убыванию). |
| 5 | История заказов клиента | Завершённые и отменённые заказы клиента (приоритет не отображается). |
| 6 | Добавить заказ | Создаёт заказ для указанного клиента. |
| 7 | Показать все заказы | Полный список заказов с именем клиента. |
| 8 | Изменить статус заказа | Устанавливает новый статус для заказа по его ID. |
| 9 | Удалить заказ | Удаляет заказ и убирает его из списка заказов клиента. |

## Модели данных

### Клиент

```json
{
    "id": 1,
    "name": "Иван Иванов",
    "phone": "+7 900 000-00-00",
    "email": "ivan@example.com",
    "orders": [1, 2]
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | int | Уникальный идентификатор (автоинкремент) |
| `name` | string | Имя клиента |
| `phone` | string | Номер телефона |
| `email` | string | Электронная почта |
| `orders` | int[] | Список ID заказов клиента |

### Заказ

```json
{
    "id": 1,
    "client_id": 1,
    "description": "Разработка сайта",
    "amount": 50000.0,
    "priority": "Высокий",
    "status": "В работе",
    "deadline": "2026-04-01"
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | int | Уникальный идентификатор (автоинкремент) |
| `client_id` | int | ID клиента-владельца заказа |
| `description` | string | Описание заказа |
| `amount` | float | Сумма заказа |
| `priority` | string | `Низкий` / `Средний` / `Высокий` |
| `status` | string | `Новый` / `В работе` / `Завершён` / `Отменён` |
| `deadline` | string | Срок выполнения в формате `YYYY-MM-DD` |

### Жизненный цикл заказа

```
Новый → В работе → Завершён
                 → Отменён
```

Переход между любыми статусами доступен вручную через пункт меню 8.

## База данных

Файл `database.json` создаётся автоматически при первом добавлении данных и хранится в директории `v1/`. Структура файла:

```json
{
    "clients": [ ... ],
    "orders":  [ ... ]
}
```

Файл исключён из git (`.gitignore`).

## Функции модуля

| Функция | Описание |
|---------|----------|
| `load_data()` | Читает `database.json`; возвращает `(clients, orders, data)` |
| `save_data(clients, orders)` | Перезаписывает `database.json` |
| `generate_id(items)` | Возвращает `max(id) + 1` или `1` для пустого списка |
| `add_client(clients, orders)` | Диалог добавления клиента |
| `show_clients(clients)` | Вывод всех клиентов |
| `delete_client(clients, orders)` | Удаление клиента и каскадное удаление его заказов |
| `add_order(clients, orders)` | Диалог добавления заказа |
| `show_orders(clients, orders)` | Вывод всех заказов |
| `show_client_orders(clients, orders)` | Активные заказы одного клиента |
| `show_client_order_history(clients, orders)` | История заказов одного клиента |
| `update_order_status(clients, orders)` | Изменение статуса заказа |
| `delete_order(clients, orders)` | Удаление заказа |
| `print_order(order, show_priority)` | Форматированный вывод одного заказа |
| `main_menu(clients, orders)` | Основной цикл меню |
